Import kuralları:
1. Şirketi company.json içinden oku.
2. Şirketi name ile bul, yoksa oluştur.
3. Şirket subdomain çakışırsa mevcut şirketi name ile kullan.
4. Tüm kayıtlar bu şirketin companyId değeriyle oluşturulsun.
5. JSON içindeki stable key alanlarını kullan:
   - unitKey
   - subUnitKey
   - energySourceKey
   - energyUseGroupKey
   - meterKey
   - variableKey
6. Database id değerlerini JSON’dan bekleme.
7. Import sırasında key → database id map oluştur.
8. Import sırası:
   - company
   - units
   - users
   - sub-units
   - energy-sources
   - energy-use-groups
   - meters
   - consumption
   - variables
   - variable-values
   - swot-items
   - risks
   - risk-notes
   - seu-items
   - energy-targets
9. Users için passwordHash JSON’dan okunmasın.
10. Import edilen demo kullanıcılar için varsayılan şifre admin123 olsun.
11. Kullanıcı username varsa duplicate oluşturma, mevcut kullanıcıyı kullan.
12. Units için aynı companyId + name varsa mevcut kaydı kullan.
13. Sub-units için aynı unitId + name varsa mevcut kaydı kullan.
14. Energy sources için aynı unitId + name varsa mevcut kaydı kullan.
15. Energy use groups için aynı companyId + name + mümkünse unit/subUnit ilişkisine göre mevcut kaydı kullan.
16. Meters için aynı companyId + name + subUnitId varsa mevcut kaydı kullan.
17. Consumption için aynı meterId + year + month varsa yeni kayıt oluşturma, mevcut kaydı güncelle veya atla. Duplicate oluşturma.
18. Boş array dosyaları sorunsuz işlensin.
19. Eksik opsiyonel alanlar hata vermesin.
20. Import sonunda özet yaz:
   - kaç company işlendi
   - kaç unit
   - kaç subUnit
   - kaç energySource
   - kaç energyUseGroup
   - kaç meter
   - kaç consumption
   - kaç user
   - kaç variable
   - kaç kayıt eklendi
   - kaç kayıt zaten vardı / atlandı
