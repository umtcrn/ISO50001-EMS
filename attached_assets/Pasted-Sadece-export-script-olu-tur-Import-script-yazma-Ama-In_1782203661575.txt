Önemli veri kuralları:

1. Database id değerlerini ana referans olarak export etme.
2. İlişkileri mümkün olduğunca stable key mantığıyla yaz.
3. Schema değiştirmediğimiz için externalKey DB'ye ekleme.
4. Export sırasında stable key üret:

   * unitKey: unit adından slug
   * subUnitKey: unitKey + subUnit adından slug
   * energySourceKey: unitKey + energy source adından slug
   * energyUseGroupKey: unitKey/subUnitKey + grup adından slug
   * meterKey: subUnitKey + meter adından slug
   * variableKey: variable code varsa code, yoksa name slug
5. Aynı isimden dolayı key çakışması olursa sonuna mevcut database id ekleyerek benzersiz yap. Ama id sadece çakışma önleme suffix'i olarak kullanılsın.
6. Consumption kayıtlarında meterId yerine meterKey kullan.
7. Users kayıtlarında passwordHash export edilmesin.
8. Users kayıtlarında unitId yerine unitKey yazılsın.
9. Sayaçlarda unitId, subUnitId, energySourceId, energyUseGroupId yerine ilgili key alanları yazılsın.
10. Tüketim verilerinde year, month, kwh, tep, co2, hdd, cdd, cost, notes gibi mevcut alanlar korunabildiği kadar korunsun.
11. Variable values kayıtlarında variableId, unitId, subUnitId, meterId yerine key alanları yazılsın.
12. createdAt / updatedAt gibi teknik alanları export etme.
